# CosyVoice-DPO: Direct Preference Optimization for Emotional TTS

This directory contains the **training-time DPO (Direct Preference Optimization)** component of [CCG-CFG-TTS](../README.md). It fine-tunes the LLM within [CosyVoice2](https://github.com/FunAudioLLM/CosyVoice) to improve emotional expressiveness by aligning the model with preferences derived from speech emotion recognition and transcription accuracy.

> For the **inference-time CFG** component (which also produces the training data for DPO), see [CCG-CFG-TTS/](../CCG-CFG-TTS/).

---

## Overview

Given a text with an emotion prompt, we construct preference pairs (chosen/rejected speech tokens) based on:
- **Speech Emotion Recognition (SER) confidence** — how well the generated audio matches the target emotion
- **Word Error Rate (WER)** — transcription accuracy of the generated speech

The DPO loss ([Rafailov et al., 2023](https://arxiv.org/abs/2305.18290)) is combined with the standard SFT cross-entropy loss to fine-tune the LLM while maintaining generation quality. IPO and label smoothing variants are also supported.

## Installation

### 1. Install dependencies

```bash
cd cosyvoice-dpo
pip install -r requirements.txt

# Install Matcha-TTS (required for config loading)
cd third_party/Matcha-TTS && pip install -e . && cd ../..
```

### 2. Download pretrained models

Download [CosyVoice2-0.5B](https://www.modelscope.cn/models/iic/CosyVoice2-0.5B) and place the files under `pretrained_models/CosyVoice2-0.5B/`:

| File | Description |
|------|-------------|
| `speech_tokenizer_v2.onnx` | Speech token extractor |
| `campplus.onnx` | Speaker embedding extractor |
| `CosyVoice-BlankEN/` | Qwen2 pretrained weights |
| `llm.pt` | LLM checkpoint (used as both init and DPO reference) |

## Data Preparation

### Generating CFG samples for DPO

DPO training requires chosen/rejected audio pairs. These are generated by the [CCG-CFG-TTS](../CCG-CFG-TTS/) component using different CFG scales and random seeds:

```bash
cd ../CCG-CFG-TTS
bash gen_audio_emotion_diff_reference_train/pipeline_cfg_gpt_condition_emotion_gt_ref.sh
```

### DPO sample selection

The selection script (`tools/select_dpo_samples_v5.py`) ranks generated samples by SER confidence and WER to form preference pairs:

```bash
python tools/select_dpo_samples_v5.py \
  --inp_gt_dir /path/to/ground_truth_metadata \
  --out_dir data/dpo_samples \
  --inp_gen_dirs /path/to/gen_seed_0_metadata /path/to/gen_seed_42_metadata \
  --disable_utmos --disable_speed_warp \
  --disable_emo_syllable_dur --disable_count_syllable_dur \
  --wers_min 0.80
```

This outputs `positive_samples.jsonl` and `negative_samples.jsonl`.

## Training Pipeline

The full pipeline is in `run_dpo.sh`. Set `stage` and `stop_stage` to control which steps to run:

| Stage | Description |
|-------|-------------|
| 0 | Select DPO training samples |
| 1 | Select DPO validation samples |
| 5 | Extract features for validation set |
| 6 | Extract features for training set |
| 7 | Combine split `.pt` files |
| 8 | Build Parquet training data |
| 10 | DPO training |

### Quick start (training only)

If you already have prepared Parquet data:

```bash
# Edit run_dpo.sh to set paths, then:
bash run_dpo.sh  # with stage=10 stop_stage=10
```

### Full pipeline

```bash
# 1. Configure paths in run_dpo.sh
# 2. Run all stages
bash run_dpo.sh  # with stage=0 stop_stage=10
```

## Training Data Format

After sample selection, the pipeline converts JSONL to Kaldi format, extracts speech tokens and speaker embeddings, and packages everything into Parquet files. Each training sample contains:

| Field | Description |
|-------|-------------|
| `text` | Input text |
| `prompt` | Emotion/style prompt |
| `speech_token` | Chosen (positive) speech token sequence |
| `reject_speech_token` | Rejected (negative) speech token sequence |
| `utt_embedding` | Utterance-level speaker embedding |
| `spk_embedding` | Speaker-level embedding |

## Key Modifications from Upstream CosyVoice2

This codebase is a fork of [CosyVoice2](https://github.com/FunAudioLLM/CosyVoice) with the following DPO-specific changes:

- **`cosyvoice/llm/llm.py`** — Added `Qwen2LM.forward_dpo()` that computes log-probabilities for both chosen and rejected speech token sequences in a single forward pass.
- **`cosyvoice/dataset/processor.py`** — Added `tokenize_with_prompt()` for emotion prompts, `sort_no_feat()` / `dynamic_batch_token()` for token-level batching, and DPO reject-token handling in `padding()`.
- **`cosyvoice/utils/losses.py`** — Added `DPOLoss` class implementing the DPO objective, with optional IPO and label smoothing.
- **`cosyvoice/utils/train_utils.py`** — Extended `batch_forward()` to compute DPO preference loss using a frozen reference model.
- **`cosyvoice/utils/executor.py`** — Extended training loop to pass reference model and DPO loss, with error recovery for numerical stability.
- **`cosyvoice/bin/train_emotion.py`** — New training entrypoint that loads the reference model and DPO loss module.
